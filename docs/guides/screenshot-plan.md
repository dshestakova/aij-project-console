# Реестр скриншотов пользовательских руководств

Скриншоты автоматически снимаются из авторизованного окружения командой `npm run docs:guides:screenshots`. Значения `GUIDE_BASE_URL`, `GUIDE_USER_EMAIL` и `GUIDE_USER_PASSWORD` берутся только из локального `.env.local` или окружения процесса и не сохраняются в репозитории.

## Правила подготовки

- Использовать тестовую или обезличенную учетную запись.
- Не показывать реальные email, ФИО, клиентов, названия проектов, комментарии, документы и ссылки на файлы.
- Не показывать адресную строку, если в ней есть внутренний домен или идентификаторы, которые нельзя публиковать.
- Не открывать инструменты разработчика, переменные окружения, Supabase или Vercel settings.
- Для экранов редактирования создать отдельный тестовый проект.
- Снимать интерфейс при ширине около 1440 px; при необходимости добавить отдельный мобильный кадр шириной около 390 px.
- Скрипт сохраняет PNG в `docs/guides/assets/` и проверяет наличие каждого обязательного кадра.
- Email учетной записи, имена CSM/директоров, автор загрузки и пользователи истории заменяются обезличенными значениями до снятия кадра.

## Проверка после обновления

1. Убедиться, что все изображения открываются из Markdown по относительным путям `assets/<имя-файла>`.
2. Проверить, что текст на скриншотах читаем при обычном масштабе.
3. Повторно проверить каждый кадр на персональные и конфиденциальные данные.
4. Убедиться, что в руководствах и PDF нет строк или блоков-заполнителей для будущих скриншотов.
5. Выполнить `npm run docs:guides:pdf` и визуально проверить итоговые PDF.
